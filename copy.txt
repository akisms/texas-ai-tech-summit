# ───────────── PROMPTING ─────────────
[prompt.title]
What separates a mediocre prompt from a great one

[prompt.1t]
Give it a role and a goal

[prompt.1d]
"You're my marketing analyst. I want X for audience Y." Context turns a generic answer into a useful one.

[prompt.2t]
Show, don't just tell

[prompt.2d]
Paste examples, data, a past piece in your voice. AI matches what it can see.

[prompt.3t]
Iterate; it's a conversation, not a vending machine

[prompt.3d]
The first answer is a draft. Refine it: "tighter," "more skeptical," "explain it for a beginner."

[prompt.4t]
Ask for the format you want

[prompt.4d]
Table, bullets, email, three options. Tell it the shape of the output up front.

[prompt.callout]
**Pro move:** stuck on how to ask? Ask Claude to write the prompt for you. "Help me write a prompt that will get me…"

# ───────────── CLAUDE CODE (vocabulary) ─────────────
[code.title]
You don't write code. You point Claude at what you want.

[code.sub]
Claude Code lives in the terminal, so it doesn't just write code: it runs it, installs what's needed, manages versions, and deploys the whole chain. You stay the decision-maker, and nothing happens without your permission. Here's the only vocabulary you need.

[code.1t]
project folder

[code.1d]
One directory on your computer that holds everything you're building. Claude works inside it, so the project stays in one place.

[code.2t]
permissions

[code.2d]
You decide what Claude can do on its own versus what needs your sign-off. Your safety dial as autonomy goes up.

[code.3t]
CLAUDE.md

[code.3d]
A plain-text house-rules file. Set them globally (how you like to work) and per project (what this build needs).

[code.4t]
GitHub & repos

[code.4d]
The cloud home for what you build. A "repo" stores your project, its history, and lets you ship it to the world.

# ───────────── GUARDRAILS ─────────────
[guard.title]
More autonomy means more responsibility

[guard.sub]
As you move up the levels, AI does more on its own. These three habits keep you in control the whole way.

[guard.1t]
Mind your data

[guard.1d]
Don't paste secrets, client-confidential info, or anything you wouldn't want stored. Know what's sensitive before you share it.

[guard.1r]
Rail: think before you paste

[guard.2t]
Verify the output

[guard.2d]
AI can sound confident and still be wrong. It "hallucinates." You finish, not it: check the facts, numbers, and claims that matter.

[guard.2r]
Rail: trust, then verify

[guard.3t]
Use permissions

[guard.3d]
The more it can act for you, the more you set boundaries. Approve big actions; let it run the safe, routine ones.

[guard.3r]
Rail: you hold the dial

[guard.seg1]
**Chat**answers only, low risk

[guard.seg2]
**Cowork**acts for you, set boundaries

[guard.seg3]
**Code**builds and runs, permission first
